OpsAgentAI

基于 Dify、RAG 与飞书机器人的 CI/CD 失败日志智能诊断实践

技术文章 GitHub 源码

项目背景

针对传统运维告警"仅通知、不诊断"的痛点，模拟 CI/CD 流水线失败后的日志分析场景。 通过整合运维文档、Dify 工作流与 LLM 能力，构建一套从失败事件捕获到修复建议推送的智能诊断原型系统。

核心职责

  • 流程设计与实现

    基于 Dify 设计 AI 诊断工作流，利用 Python 开发中转网关，对接 GitHub Actions Webhook，实现失败日志的自动化清洗与上下文提取。

  • 知识库（RAG）实践

    将常见构建失败、依赖异常、脚本执行错误等运维文档整理为知识库，验证 LLM 在给定上下文下生成排查建议的可行性。

  • 通知闭环集成

    集成飞书机器人消息，设计"失败步骤-可能原因-修复建议-相关文档"的推送格式，方便运维人员快速复核排查结论。

  • 工程化处理

    设计日志截断、关键错误提取和降级返回逻辑，避免长日志直接进入模型导致 Token 浪费，并保留人工确认环节。
