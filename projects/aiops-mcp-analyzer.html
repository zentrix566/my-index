AIOps MCP Analyzer

把服务器巡检、MCP 证据补查、规则分析、DeepSeek 归因和 Web 可视化串成一条运维诊断链路

GitHub 源码 技术文章

项目背景

传统服务器告警通常只告诉你“哪里红了”，但真正耗时的是继续补查 CPU、负载、磁盘、inode、I/O、进程和日志等证据。 AIOps MCP Analyzer 将这些补查动作工具化，并结合规则分析与 LLM 归因，把告警排查从单点脚本升级成可复用、可解释、可接入 Agent 的诊断服务。

核心能力

  • 服务器指标巡检

    采集主机负载、CPU、内存、磁盘、inode、磁盘 I/O、Top 进程与大日志文件，快速还原服务器当前状态。

  • MCP 证据补查

    通过 FastMCP SSE 暴露运维工具，让 Agent 可以按需调用 get_host_cpu、get_host_disk、get_top_processes 等诊断能力。

  • 规则分析与 LLM 归因

    先用规则给出处置建议，再接入 DeepSeek 生成根因摘要；当 LLM 调用失败时自动回退，保证诊断链路不断档。

  • Web/API 双入口

    FastAPI 提供 HTTP API 和静态 Web 页面，前端集中展示处置建议、LLM 结论、MCP 补查证据、相关性与置信度。

工程亮点

🔎

证据链清晰

每条诊断建议都能回看 MCP 补查证据，减少纯黑盒结论带来的不确定性。

📈

相关性评分

为 evidence 计算 0-100 分相关性，区分强证据、一般证据和低相关信息。

✅

置信度输出

综合证据覆盖率、平均相关性、规则结论与 LLM 置信度，给出最终诊断可信度。

部署与使用

  • Web/API 服务

    默认监听 http://0.0.0.0:8080，可通过 /api/analyze/full 发起全量分析，也可以直接访问 Web 页面查看结果。

  • MCP 服务

    默认监听 http://0.0.0.0:8000/sse，适合接入支持 MCP SSE 的客户端或运维 Agent。

  • 生产托管

    README 中提供 API 与 MCP 两个 systemd service 示例，方便部署到 Linux 服务器长期运行。
