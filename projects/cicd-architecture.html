CI/CD 流水线体系实践

围绕 GitHub Actions、Jenkins、Kubernetes 发布模式整理的 DevOps 实践项目

GitHub 源码 技术文章

项目背景

这个项目用于梳理 DevOps 流水线的关键环节：代码提交、质量检查、镜像构建、制品推送、Kubernetes 发布、失败通知和发布后观测。 通过多个实践仓库验证 GitHub Actions、Jenkins、Docker、K8s 发布模式与飞书通知的组合方式，为后续进入运维平台/DevOps/AIOps 岗位补齐工程化实践。

实践内容

  • 流水线模板整理

    整理 GitHub Actions 与 Jenkinsfile 示例，覆盖代码拉取、测试、构建镜像、推送镜像仓库、远程触发部署等常见流程。

  • 容器化与镜像交付

    编写 Dockerfile 与 Nginx 配置，验证静态站点容器化、镜像构建、镜像推送和 Kubernetes 中滚动更新的基本链路。

  • 质量门禁实践

    学习并验证 JUnit、JaCoCo 等质量检查工具在流水线中的接入方式，理解测试失败、覆盖率不达标时的阻断思路。

  • Kubernetes 发布模式研究

    整理滚动发布、蓝绿发布、金丝雀发布、A/B 发布的配置与回滚思路，理解不同发布策略的适用场景和风险点。

  • Jenkins 扩展探索

    通过 Jenkins 插件实践学习插件结构、触发器配置和多模块构建触发逻辑，为后续运维平台集成打基础。
